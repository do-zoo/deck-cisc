04 / 22PAIN POINTS
The Problem We're Tackling

Setiap matchday, jam habis
buat administrasi — bukan buat anggotanya.

01 · PENDAFTARAN

Daftar nobar masih lewat japri WhatsApp satu-satu. Rekap pakai spreadsheet yang gampang ketuker antar pengurus.

02 · VALIDASI MEMBER

Nomor anggota dicek manual ke direktori Excel. Sering keliru, sering double-booking di event yang sama.

03 · BUKTI BAYAR

Screenshot transfer numpuk di galeri HP pengurus. Susah lacak, susah audit, susah dilampirkan ke laporan.

04 · LAPORAN

Tutup buku per event makan waktu berhari-hari. Angka kehadiran, finansial, menu — semuanya direkap manual.

05 / 22STRATEGY
Our Game Plan

Empat target yang harus
menang malam ini.

  1. G1Pangkas waktu admin per event — pengurus balik fokus ke pengalaman anggota, bukan ke rekap.
  2. G2Akurasi data anggota — satu sumber kebenaran, no more double-booking dan member ghost.
  3. G3Pengalaman pendaftaran < 3 menit dari HP — buat member dan tamu, tanpa instalasi apa pun.
  4. G4Jejak audit lengkap — siapa setuju, kapan, kenapa — semua tercatat untuk akuntabilitas pengurus.

TECH STACK · BACK-OFFICE
08 / 22UNDER THE HOOD
Behind the Scenes

Stack yang dipilih biar pengurus berikutnya gampang lanjutin.

RUNTIME

Next.js 16

App Router, Server Actions, React 19. Satu codebase untuk web publik & admin.

DATABASE

Postgres · Neon

Serverless Postgres dengan branching. Prisma ORM via adapter, pooled connection.

AUTH

Better Auth

Magic link + email/password, role gating untuk Owner / Admin / Verifier / Viewer.

STORAGE

Vercel Blob

Bukti transfer dan foto kartu di-convert ke WebP via Sharp sebelum upload.

UI

shadcn · Base UI

Komponen yang konsisten, Tailwind utility, React Hook Form + Zod untuk validasi.

HOSTING

Vercel

Zero ops, preview deployment per branch. Migrate jalan otomatis tiap build.

FIRST HALF · PUBLIC FLOW
10 / 22USER JOURNEY
First Half · Peserta Daftar Event

Enam touch, satu kali main.
Selesai di HP < 3 menit.

01

Lihat Fixture

Homepage menampilkan event aktif. Kartu hanya muncul kalau jendela registrasi masih buka.

02

Buka Detail

Slug-based URL dengan harga, jadwal, kebijakan operasional, dan ringkasan biaya transparan.

03

Isi Form

Data diri, pilihan tiket, partner (jika layak), pilihan menu — validasi bahasa Indonesia.

04

Upload Bukti

Bukti transfer + foto kartu anggota. Auto-convert ke WebP, max 1600px, quality 80.

05

Submit

Transaksi Prisma + upload Blob. Roll back otomatis kalau ada langkah yang gagal.

06

Konfirmasi

URL unik per registrasi. Status awal terkirim → menunggu tinjauan admin.

SECOND HALF · ADMIN INBOX
13 / 22VERIFICATION
Second Half · Admin Workflow

Tiap pendaftaran masuk inbox. Verifier ambil keputusan dengan jejak.

  1. A1Inbox per event — daftar registrasi dengan filter status & urut prioritas peninjauan.
  2. A2Detail registrasi — profil pendaftar, snapshot harga, bukti bayar, riwayat keputusan.
  3. A3Aksi tinjauan — Approve · Reject · Payment Issue, lengkap dengan template WhatsApp.
  4. A4Guard server-side — guardEvent() setiap action; tidak pernah ada role check ad-hoc.
  5. A5Audit log — siapa, kapan, kenapa — semua tercatat di histori registrasi.
